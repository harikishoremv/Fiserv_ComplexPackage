								<h2><a id="Language-Contet"></a>Language Content</h2>
								<p>This site includes a language selector on each page that controls the language that is visible at any given time. The enhancement is programmed to detect the users default language and the enhancement will automatically load the content relating to that language upon initial visit to the site. The user is always able to switch languages on any page at any time and the content will switch to the chosen language and keep that selection saved on the visitors machine to apply when the site is visited again.</p>
								<p>The enhancement uses a series of preset classes to apply lang attributes to sections of content within the site. This enables search engines and screen readers to identify and index content appropriately as well as for the selected language content to show to the visitor. Language classes are case sensitive. Use the CMS drop down to apply the class.</p>
								<p>The CMS is set up to show a small language indicator box in the upper left corner of the section as well as outline the area that is contained within that language class. These markings will not show up on the main site. They are for your reference and to assist when formatting or editing content.</p>
								<h3>Subsection Tagging</h3>
								<p>The site is currently set up using a single cell table that wraps all of the content with a particular language together. There is then a language class that is applied to the wrapper table. All language versions of the page are in the same article.</p>
								<p>If a section shows for ALL languages, do not apply a wrapper table or any language class to that section. This will allow the same section to show on ALL language versions of the page.</p>
								<h3>Element Tagging</h3>
								<p>Elements, headlines, paragraphs, images, lists, tables, etc can also be classed with language classes to control their view. Only use element tagging if the outer most table or wrapper is not being used with a language class.</p>

				<p>The CMS is set up to allow you to create and link* to new pages as needed. Within the CMS pages on your website are referred to as 'Articles' and it is possible to have more than one article per page. A good example of this is the homepage.</p>

				<h3>Creating Links</h3>
				<p>You can create a link to another page, a document, or an outside website from within the editor.</p>
				<h4>Linking to other pages</h4>
				<p>To create a link, first type the text you would like to be linked, highlight the text, then choose the 'Insert/Edit Link' from the icon menu above.&nbsp;&nbsp;</p>
				<p>Within the dialog box,&nbsp;use the article name with dashes in place of spaces for the URL field in the 'Insert/Edit Link' dialog box.&nbsp; For example, an article named 'Business Online Banking' would be '<a href="Business-Online-Banking">Business-Online-Banking</a>' when used in the URL field.&nbsp;&nbsp;You may also adjust the text that will be displayed and add classes as appropriate from the dialog box.</p>
				<p>Refer to the <a href="#Buttons">Adding Classes section > Buttons</a> to create a button link.</p>
				<h4>Linking to Documents</h4>
				<p>When linking to documents, you must first insert the document into the editor.&nbsp; To do this, choose the 'Insert' drop down option from the menu above and choose 'Insert Document.'&nbsp; This will show a list of all the documents that have been uploaded to the CMS in alphabetical order.&nbsp;&nbsp;</p>
				<p>Once you have chosen a document, highlight the text, then choose 'Insert/Edit Link' from the icon menu.&nbsp; Here you can change the text to be displayed.&nbsp; It is also recommended to have documents open in a new window, which you can adjust using the 'Target' drop down box.</p>
				<h4>Anchor Tags</h4>
				<p>You can place what is called an 'Anchor Tag' within your content that indicates a specific landing spot on the page.&nbsp; This could be used when linking from another page to go to a specific checking product or rates table. These are best placed with Heading tags.&nbsp; To include this feature, place your cursor at the beginning of the Heading and choose 'Insert' &gt; 'Anchor' from the menu above.&nbsp; The name should be short and descriptive and should not contain spaces.</p>
				<p>When linking to the anchor, you will use the pound/hashtag symbol before the name of the anchor in the URL field in the 'Insert/Edit Link' dialog box. <em>EX:&nbsp; #Anchor-Tags</em> If you are linking from a different page than the one the anchor is located in, you would put the page name first. <em>EX: Sample-Style#Anchor-Tags</em>. Please note:&nbsp; Anchor tags are case sensitive so make sure the anchor and the link match exactly.</p>

						<p>While some mobile browsers will automatically detect phone numbers and convert them to links, we recommend always adding hyperlinks to be safe.  By default, your website will visually hide the styling of the link for desktop users and only display the link on a mobile device.</p>
						<h4>Example link formatting</h4>
						<p>To mark a phone number as a link, use <strong>tel:</strong> followed by the number using the format below, within the URL field of the link popup box:</p>

								<li><a href="tel:+13335555555">333-555-5555</a></li>
							</ul>							
						</ul>
						<h4>What does the +1 mean in front of the phone number?</h4>
						<p>The plus sign “+”, signifies the international dialing prefix, "1" indicates the country code, in this case USA.</p>
						<p>Having the number with the "+" format allows you to call that number worldwide without having to remember the unique international numerical dialing prefix.</p>

<h2>Site Notice</h2>
<p>Site notices are areas of the website where alert or notice type of information can be displayed. These notices can be minimized or dismissed by the user but will reappear if the content of the message is changed. This message can be modified within the Content Manager in the "Site Notice" article usually located in the Misc content group. </p>
<p>Notice types have been developed and styled specifically for this design. To see available notice types or edit a current notice open the "Site Notice" article and click on the table, right click and choose "Table Properties". Open the class dropdown and look for class names starting with "Table-Notice" to identify the notice types available. These classes can be applied to this table to change the appearance of the site notice. A unique id must be included on the table. The background color can also be changed by adding a hexadecimal code to the "Background Color" field. If a background color is not set, the default override styled for this design will be used.</p>
<p>A site notice is enabled and will show on the site if this "Site Notice" article contains content. To disable the site notice from showing on the site, simply remove all contents of this article. Do not delete the entire article through the configure action. To enable a site notice after it has been disabled, open the "Site Notice" article and click the "Table" menu item, hover over "insert table" and choose a 1x1 table. Apply an available "Table-Notice" class and a unique id to this table using the "Table Properties" edits mentioned above.. Place your cursor inside of this table and type the text to appear in the notice.</p>
<h3>Tips</h3>
<ul>
<li>Limit subjects of site notices to one topic per notice.</li>
<li>Avoid using inline styling such as changing text sizes within the notice.</li>
<li>Avoid placing images in notices.</li>
</ul>

				<p>Create a Quicklink Section to implement striking call-to-action links for key areas of your site.  This is normally placed on the homepage but can be added to any page.</p>
				<p>The sample above uses an outer table with class "Subsection-Table-Quicklinks". This is only needed if you want a separate section to hold the content.  All that is required is the inner table with the class "Table-Quicklinks-Grid".  This "Table-Quicklinks-Grid" table can be placed in the content area of a section just like any other table. </p>
				<p>To apply an icon to a link, simply create a link with the text you would like and then click into the link, right click > add class and choose the appropriate icon class.</p>
				<p>If you would like a heading with a supporting description like the second example above, follow the steps above to apply an icon to a link and using your cursor select the text you would like as the heading and apply <strong>bold</strong> to it.  This will automatically increase the size and give it the heading look.</p>
				<h3>Helpful Tips</h3>
				<ul>
				<li>We encourage you to keep the quicklinks links consistent.  They should all be a short name or a heading with a description</li>
				<li>If you have a heading with a description present and change the link, the formatting will be lost.  You will have to re-apply the <strong>bold</strong> to the text you would like as the heading. If the content is too squished, you can add the class "Table-Quicklinks-Grid-Description" to the inner table to increase the size of the table cell.</li>
				<li>Only add one link per table cell (td).</li>				
				</ul>
